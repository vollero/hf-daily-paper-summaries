# ReflectWorld-MM: An Entity-Oriented Multimodal Memory System for Open-Ended Video Streams

> [← Daily summary](../2026-07-21.md) · [Relationship graph](relationships.md) · [Hugging Face](https://huggingface.co/papers/2607.09759) · [arXiv](https://arxiv.org/abs/2607.09759)


## One-sentence takeaway

ReflectWorld-MM externalizes streaming audiovisual experience into persistent entity-linked episodic, editable semantic, and procedural memories, achieving the highest reported accuracy among compared systems on six long-video and lifelong-memory benchmarks while usually avoiding source-video fallback.

## Problem and motivation

The paper addresses assistants that must continuously ingest audiovisual streams, preserve observations over potentially unbounded time, and later answer questions or respond to events. Existing streaming-video systems typically retain frame- or token-oriented state inside model context or memory caches, making them poorly suited to persistent entities and open-ended operation; text-agent memories provide explicit episodic, semantic, and procedural stores but do not perceive video or resolve visual identities (Sections 1, 2.1–2.2).

The authors identify a further gap in explicit multimodal memory: M3-Agent lacks substantially revisable semantic knowledge, while WorldMM lacks persistent entity identities and uses dataset-specific temporal scales. Neither combines multi-scale episodic, evolving entity-centric semantic, and procedural memory as a deployable live-stream service (Sections 1 and 2.3).

## Approach and contributions

- **Entity-resolved perception:** Activity-coherent audiovisual segments are processed using detection and face/body re-identification evidence. A dedicated resolver alone commits persistent identities, separating evidence generation from identity decisions (Sections 3.1–3.2; Algorithm 1).

- **Context-enhanced, adaptive perception:** Before interpreting each segment, the VLM receives bounded event working memory, per-camera scene context, recognized-entity history, and optionally agent-provided focus directives. Deterministic and semantic policies vary processing effort while guards preserve critical evidence (Section 3.2; Appendix C).

- **Hierarchical external memory:** Episodic memory contains entity observations, segment-level traces, and chapter-level schemas. Semantic memory periodically consolidates entity knowledge through Add, Update, Delete, or no-change operations; procedural memory holds user rules and preferences (Sections 3.3–3.5).

- **Tool-based retrieval and deployment:** An answering agent searches selected memory levels, expands temporal neighbors, and can inspect associated key frames. The persistent database service supports multiple live-source adapters and off-the-shelf assistants (Section 3.6; Appendices D–F).

- **Evaluation:** The system is evaluated across six general, egocentric, and entity-centric video benchmarks, with answer-time component ablations and answer-efficiency measurements (Section 4).

These results support the authors’ claim that the constructed memory can improve QA—especially in identity-sensitive settings—and often supplies enough evidence without reopening the source video. They do not establish indefinite operation under measured production load, prove that entity orientation alone causes the gains, or provide a fully controlled comparison across every baseline: several results are quoted from earlier papers, and some M3 baseline rows used a different judge (Appendix A).

## Limitations and validity risks

- **Author-stated limitation:** Absolute M3-bench accuracy remains low for all systems, so entity-centric, multi-hop memory reasoning is not solved (Section 4.2).

- **Author-stated limitation:** Ablations are performed only at answer time on two entity-sensitive benchmarks; memory is not rebuilt without each component. The authors characterize resulting drops as lower bounds because overlapping evidence remains accessible (Section 4.4).

- **Analyst inference — comparison consistency:** Baselines combine author re-runs and numbers quoted from other papers. In particular, older M3-bench baselines used a GPT-4o judge, whereas GPT-5, M3-Agent, and ReflectWorld-MM use GPT-5-mini, limiting direct comparability beyond the consistently re-evaluated rows (Appendix A; Tables 4–5).

- **Analyst inference — causal attribution:** ReflectWorld-MM changes several elements simultaneously—perception context, entity resolution, memory hierarchy, consolidation, retrieval, and visual fallback. The limited answer-time ablation cannot isolate their effects during memory construction.

- **Analyst inference — open-ended efficiency:** The design claims bounded per-segment working state and query context, but the supplied evaluation reports no throughput, latency, database-growth, storage, or long-duration stability measurements (Sections 3.1, 3.6, and 4.6).

- **Analyst inference — model and judge dependence:** Experiments depend on GPT-5, GPT-5-mini, and OpenAI embeddings, including an LLM judge for open-ended answers. No judge-agreement or sensitivity analysis is reported (Section 4.1; Appendix C).

- **Analyst inference — fallback complicates memory-only claims:** A visual-evidence tool can fetch source key frames, and 4.6–6.8% of questions require video fallback in the reported efficiency tests. Thus, aggregate accuracy is not purely memory-text retrieval performance (Table 3; Appendix D).

## Open research questions

1. **How much does entity resolution improve memory construction rather than retrieval alone?** The current ablation blocks entity-linked retrieval after full memory has already been built. A stronger experiment would rebuild memory with entity resolution disabled or deliberately perturbed and measure QA, identity consistency, and semantic-fact errors.

2. **Does the system remain efficient and accurate over genuinely long-running streams?** The architecture is designed for unbounded operation, but no duration, storage-growth, latency, or drift curve is supplied. A multi-month continuous-stream study could track per-segment latency, database size, retrieval cost, identity errors, and QA accuracy over time.

3. **How reliable are semantic updates and deletions?** The paper gives a qualitative week-long example but no quantitative evaluation of fact addition, revision, deletion, or contradiction handling. A temporally changing entity-fact benchmark could measure precision, update delay, stale-fact retention, and erroneous deletion.

4. **How robust are conclusions to the answering model and evaluation judge?** GPT-5 is both the answer agent and an important reference, while GPT-5-mini judges M3 outputs. Repeating evaluation with multiple answer models, independent judges, and human adjudication would test whether gains persist across model and scoring choices.
