# LongStraw: Long-Context RL Beyond 2M Tokens under a Fixed GPU Budget

> [← Daily summary](../2026-07-17.md) · [Relationship graph](relationships.md) · [Hugging Face](https://huggingface.co/papers/2607.14952) · [arXiv](https://arxiv.org/abs/2607.14952)


## One-sentence takeaway

LongStraw fits GRPO-shaped response replay at up to 4,456,448 positions on eight H20 GPUs by detaching and compactly storing prompt state, then serializing short response graphs, but the reported paths do not establish globally correct distributed updates or useful policy learning.

## Problem and motivation

Long-context inference is approaching million-token inputs, while RL post-training often remains at 256K tokens or less because training must retain or reconstruct backward state for multiple responses sharing a long prompt. This is particularly limiting for agents whose observations, tool outputs, documents, and decisions accumulate over long trajectories (Abstract; Section 1).

The authors identify a gap not fully addressed by efficient attention, parameter-efficient tuning, checkpointing, or scale-out sequence parallelism: making grouped RL execution fit while holding GPU count fixed. The main bottlenecks are the lifetime and physical ownership of prompt state, response graphs, distributed communication buffers, and optimizer state—not merely attention computation (Sections 1 and 3.4).

## Approach and contributions

- **Detached prompt capture and serial suffix replay:** LongStraw evaluates the shared prompt once without autograd, stores only architecture-required conditional state, and replays one response branch at a time under autograd. This bounds the live policy graph by the largest response branch rather than the prompt-plus-response sequence or all group members (Sections 2.1, 4.1; Figures 2 and 4).

- **Architecture-specific state management:** For Qwen3.6-27B, it retains compact recurrent GDN state and physically compact CP8-sharded KV pages on GPUs. For GLM-5.2, it places CP32-sharded MLA latent and DSA index-key pages on CPUs and stages one decoder layer at a time (Sections 3–6; Tables 1–3).

- **Memory-oriented execution techniques:** The implementation combines right-sized physical pages, whole-layer response checkpointing, serial group scheduling, context/expert parallel ownership, and LoRA/QLoRA adaptation (Sections 4.2–4.6).

- **Explicit evidence hierarchy:** The paper distinguishes execution capacity, response-operator fidelity, distributed-update consistency, and full-gradient parity, and audits which levels each implementation reaches (Section 2.3; Table 8).

- **Fixed-budget execution receipts:** It reports Qwen paths at 2,097,152 and 4,456,448 positions on eight H20s and a GLM path with a 2,097,152-token prompt on 32 H20s, while explicitly documenting missing forward or gradient collectives (Sections 5, 6, 8, 9.7; Appendix F).

The evidence establishes that architecture-aware state capture and serial replay can execute unusually long, GRPO-shaped workloads within the stated GPU allocations. It also supports near-flat Qwen peak allocation between the two measured group sizes.

It does **not** establish conventional full-sequence gradient equivalence, coherent distributed updates, general group-size scaling, cost or throughput superiority, useful behavior at these context lengths, or policy improvement. The 4.25M Qwen results additionally use either a replay-only path without an optimizer or a modified prefix-frozen objective.

## Limitations and validity risks

- **Author-stated — detached prompt gradients:** Both implementations omit the gradient through prompt-state construction, corresponding to the second term in Equation 3. Full-sequence gradient parity is therefore not claimed (Sections 2.2 and 12.3).

- **Author-stated — incomplete distributed updates:** Qwen does not compose all K/V-derived adapter and upstream gradients across CP8. The historical GLM path skips gradient finalization for CP-replicated non-expert adapters (Sections 5.6, 6.8, and 12.1).

- **Author-stated — GLM forward mismatch:** Historical GLM attention is local to each 65,536-token shard rather than global over the 2,097,152-token prompt (Sections 6.7 and 12.2).

- **Author-stated — execution probe rather than RL evaluation:** Responses are supplied and synthetic, rewards deterministic, and no generation, environment interaction, reward-model execution, repeated standard updates, or policy-quality evaluation is included (Sections 2.1 and 12.4).

- **Author-stated — inactive objective components:** Qwen uses \(\beta=0\), identical old/reference scoring, and an unclipped live ratio term. GLM has \(\epsilon=0.2\) and \(\beta=0.01\), but its initial ratio is one and KL zero, so clipping and KL are inactive (Sections 5.4, 8.2–8.3).

- **Author-stated — incomplete resource accounting:** There is no matched baseline, full host/network/energy/cost accounting, timing variance, or valid GLM whole-transaction peak. Qwen group scaling is measured only at \(G=2\) and \(G=8\) (Section 12.5).

- **Analyst inference — limited causal attribution:** Because several mechanisms are combined and the paper does not provide controlled end-to-end ablations of each one at the principal settings, the individual contribution of page compaction, checkpointing, offload, and serialization to final capacity is not precisely isolated. The staged GLM failure ladder provides diagnostic evidence, but not a matched ablation study (Table 4; Sections 7.1–7.7).

- **Analyst inference — reproducibility remains constrained:** The released stack is identified prospectively, but the historical 2M GLM receipt is not bound to that exact source stack, and the canonical artifact omits several run identifiers and software details (Section 8.1; Tables 6 and 10; Section 12.5).

## Open research questions

1. **Can LongStraw produce a coherent distributed conditional-response update?**  
   This remains open because Qwen lacks selective K/V-related reductions and historical GLM skips CP gradient finalization. A next experiment should repair these paths and compare every LoRA gradient shard and optimizer delta against a short-context distributed reference.

2. **How close is detached-state replay to full-sequence training?**  
   Equation 3 shows that prompt-state gradients are omitted, and no numerical parity test is reported. A plausible experiment is a 32K–64K matched run comparing loss, per-parameter gradients, optimizer deltas, and subsequent logits between conventional autograd and replay.

3. **Does globally faithful GLM DSA remain feasible at 2M or beyond?**  
   The current code reportedly adds global candidate and output composition, but only a 32K forward canary covers it. A fresh 2M run should test layerwise forward parity, whole-transaction memory, communication cost, and backward synchronization.

4. **Does the execution method improve real long-horizon policies?**  
   The paper evaluates synthetic supplied trajectories rather than online RL. A next experiment should perform rollout, reward computation, frozen old-policy scoring, update, checkpoint/reload, and repeated evaluation on a long-context agent task.

## Confidence and missing information

**Confidence: high** for the paper’s execution-capacity claims and their stated boundaries, because the supplied text contains detailed timings, memory accounting, trace inventories, and an explicit evidence matrix.

Material missing information includes:

- policy-quality or downstream long-context task results;
- a conventional full-sequence gradient-parity result;
- corrected 2M receipts for global GLM DSA and finalized gradients;
- coherent Qwen CP8 post-step parameter checks;
- a valid GLM whole-transaction memory peak;
- matched baselines and repeated-run variance;
- broad context-length and group-size sweeps;
- complete host-memory, communication, utilization, energy, and cost accounting;
- several canonical GLM run details, including the actual learning rate, token checksum, LoRA seed, final model revision, and full software/NCCL versions.
